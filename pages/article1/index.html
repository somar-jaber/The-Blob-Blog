If you mainly use .md files but want to leverage Nunjucks functionality in Eleventy, here is how you can do that:

Enable Nunjucks for Markdown files

In your Eleventy config file (.eleventy.js), set the Markdown template engine to Nunjucks. This way, Eleventy will first process your Markdown files with Nunjucks, enabling you to use Nunjucks syntax inside .md files:

module.exports = function(eleventyConfig) {
  return {
    markdownTemplateEngine: "njk",
  };
};

How it works

  • Your .md files will still be parsed as Markdown, but before that, Nunjucks will render any Nunjucks tags or expressions inside.
  • You can use Nunjucks features like includes, shortcodes, control structures, variables, filters, etc., directly inside your Markdown content.
  • Your Markdown front matter can specify layouts that are .njk files, enabling rich templating with reusable layouts.
  • For example, you can use this front matter in your .md file:
---
layout: base-layout.njk
---

# Hello Eleventy

This will use Nunjucks to set the variable and then render Markdown headings, combining both seamlessly.

Additional option: override template engine per file

If you want to enable Nunjucks selectively rather than globally for all Markdown files, add this to the front matter of individual .md files:

---
templateEngineOverride: njk,md
---

This tells Eleventy to process the file first with Nunjucks, then with Markdown.

Summary

  • Set markdownTemplateEngine to "njk" in your Eleventy config for global support.
  • Use Nunjucks code inline in your Markdown.
  • Combine Nunjucks layouts (.njk) with .md content easily.
  • Use the templateEngineOverride front matter for per-file control.

This approach harnesses the full power of Nunjucks templating inside Markdown files, giving you the flexibility to create dynamic, modular content in Eleventy[web[2][3][4]
